Interface IWorkQueue<TItem>

Interface IWorkQueue holds a series of work item objects. When objects are removed from the queue they are noted as well in a processing set. If new items arrive while processing they are not added to the queue until the processing of that item is Done(TItem). In this way processing the same item twice simultaneously due to incoming event notifications is not possible. Ported from https://github.com/kubernetes/client-go/blob/master/util/workqueue/queue.go.
